BLOCKFORGE 5.6 · STABILITY & RELEASE EDITION

Architecture

A browser-only, offline-first structural sandbox. No account, server database, Firebase or Supabase is required.

Runtime flow

HTML shellCore catalogGame stateRenderingFeature modulesLocal persistencePWA cache

Core engine

core.js owns constants, materials, tools, translations, state and grid creation. game.js handles simulation, rendering, objectives and scoring. ui.js binds controls.

Progressive modules

enhancements.js, local-suite.js, refined.js, portfolio-edition.js and portfolio-modes.js progressively enhance the same stable core.

Persistence

Scenes, settings, profiles, replay, diagnostics and migrations use versioned localStorage keys. User-controlled JSON exports provide portable backups.

Offline layer

sw.js precaches the landing, game, documentation, test pages and runtime assets. The manifest launches play.html as a standalone PWA.

Accessibility

Keyboard focus, skip links, reduced motion, contrast, top/flat views, textual scene descriptions and speech output are layered independently from gameplay.

Stability layer

stability.js adds runtime audits, error logging, diagnostics export, schema migration, performance measurements and low-power mode.

Scene data model

{
  id: number,

  hp: number

Rendering strategy

The scene is rendered with DOM elements and CSS 3D transforms. Camera state is represented by spin, tilt and zoom. Heavy visual effects are reduced when low-power mode or reduced-motion preferences are active.

Known trade-offs

The structural model favors explainability and fast browser execution over engineering-grade finite element simulation. Media recording, speech, vibration and gamepad support depend on browser capabilities.
