BlockForge 5.5 Documentation

Technical and user documentation for the fully local Portfolio Edition.

Architecture

The game uses a layered browser architecture: core.js defines state and materials, game.js handles rules and physics, ui.js binds controls, while enhancement modules add PWA, Local Lab, Engineer View, version history and Portfolio Edition systems.

Physics

Structures are represented on an 8×8×8 grid. Stability considers vertical support, material mass, column load, gravity and experimental wind or earthquake effects. Engineer View exposes pressure, center of mass and likely collapse zones.

Storage and privacy

All progress is stored locally with localStorage. No account, Firebase, Supabase or cloud database is used. Users can export scenes, replays, reports and complete backups.

Accessibility

Keyboard navigation, skip links, contrast options, reduced motion, scene descriptions, speech output, top view, flat view and configurable touch controls are included.

PWA and offline

The service worker caches the landing page, game, documentation and major modules. The installed application opens the game directly.

Controls

Ctrl/⌘+K opens Command Palette. G toggles gravity. Ctrl/⌘+Z performs undo. Escape closes overlays.

Quality

Use Quality Gate for smoke tests and Known Issues for browser limitations.
